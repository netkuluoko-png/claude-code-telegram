Створи кістяк для нового окремого MCP-сервера на Python. Його назва: `mcp-scheduler`.
Цей сервер буде відповідати за планування та керування відкладеними/регулярними завданнями для агента. 

Для збереження стану завдань використовуй SQLite (можна aiosqlite).

Сервер повинен експортувати ТРИ інструменти (tools):

1. `schedule_task`: Створює нове завдання.
Параметри: 
- `task_name` (str, обов'язково)
- `prompt` (str, обов'язково) - контекст і задача для агента.
- `schedule_type` (Literal["once", "interval", "cron"], обов'язково)
- `run_at` (str, ISO 8601, опціонально, вимагати якщо type="once")
- `interval_minutes` (int, опціонально, вимагати якщо type="interval")
- `cron_expression` (str, опціонально, вимагати якщо type="cron")
- `max_runs` (int, опціонально) - ліміт запусків.

2. `list_tasks`: Повертає список усіх активних завдань.
Параметри:
- `status_filter` (str, опціонально) - наприклад "active", "completed". Якщо не передано - повертає всі активні.
Повинен повертати ID завдання, назву, тип розкладу та час наступного запуску.

3. `delete_task`: Скасовує та видаляє завдання з розкладу.
Параметри:
- `task_id` (int/str, обов'язково) - ID завдання для видалення.

Що потрібно зробити прямо зараз:
1. Ініціалізуй базовий клас MCP-сервера.
2. Напиши Pydantic-схеми для параметрів цих трьох інструментів із детальними descriptions, щоб я (як LLM) розумів, як їх використовувати.
3. Напиши код для ініціалізації SQLite бази (таблиця `tasks`) при старті сервера.
4. Зроби заглушки (або базову реалізацію) для обробників цих трьох інструментів, які поки що просто пишуть/читають з бази.
Саму логіку воркера/планувальника (який буде реально запускати ці таски) напишемо наступним кроком.
